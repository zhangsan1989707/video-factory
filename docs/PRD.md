## 2. 背景与目标

当前 `video-factory` 已具备 AI 脚本生成、分镜规划、TTS、PIL/HyperFrames 渲染、ffmpeg 合成、Web 控制台和 CLI 等基础能力。现有能力主要围绕 GitHub 项目介绍和热榜视频。

新功能目标是在不破坏现有 GitHub 视频管线的前提下，新增一个独立的“炒股科普”内容域，让用户输入一个股票知识主题，系统自动生成一条 60 秒竖屏科普视频。

核心目标：

1. 让非专业剪辑用户可以快速生成“股票知识点科普”视频。
2. 建立固定的金融科普脚本结构，避免 AI 自由发挥导致内容松散。
3. 建立内容合规边界，明确不荐股、不预测涨跌、不承诺收益。
4. 复用现有 TTS、视频合成、任务产物管理和 Web 控制台能力。
5. 后续可以扩展到基金、ETF、仓位管理、交易心理等内容。

## 3. 产品定位

### 3.1 一句话定位

面向股市新手和初级散户的 60 秒股票知识科普视频生成工具。

### 3.2 产品不做什么

本功能不做以下内容：

- 不生成具体个股买入建议。
- 不生成具体个股卖出建议。
- 不生成明日涨跌预测。
- 不生成收益承诺、翻倍暗示、稳赚话术。
- 不生成“推荐股票池”“牛股推荐”“黑马股”等荐股内容。
- 不直接接入实时行情做交易决策。

### 3.3 产品允许做什么

本功能允许：

- 解释股票技术指标概念。
- 解释常见交易术语。
- 讲解历史案例或模拟案例。
- 讲解新手常见误区。
- 讲解交易纪律和风险控制。
- 展示虚拟 K 线、虚拟指标曲线、示意图。
- 给出学习建议和风险提醒。

| 用户类型 | 特征 | 内容需求 |
|---|---|---|
| 股市新手 | 刚开户或刚开始看盘，术语不熟 | 大白话解释、少术语、强类比 |

## 5. MVP 范围

### 5.1 MVP 必做功能

| 编号 | 功能 | 说明 | 优先级 |
|---|---|---|---|
| F1 | 炒股科普入口 | Web 控制台新增“炒股科普”页面 | P0 |
| F2 | 主题输入 | 输入视频主题，如“60 秒带你搞懂 MACD” | P0 |
| F3 | 参数选择 | 选择主题类型、视觉风格、目标用户 | P0 |
| F4 | AI 生成口播稿 | 生成 60 秒中文口播脚本 | P0 |
| F5 | AI 生成分镜 | 生成 7 段式短视频分镜 | P0 |
| F6 | 合规检查 | 检查荐股、预测、收益承诺等风险话术 | P0 |
| F7 | 分镜预览 | 展示每个镜头的标题、字幕、口播、模板类型 | P0 |
| F8 | 编辑脚本 | 支持人工编辑标题、口播、字幕 | P0 |
| F9 | TTS 生成 | 复用现有 edge-tts 生成语音 | P0 |
| F10 | HyperFrames 渲染 | 生成 1080x1920 竖屏视频 | P0 |
| F11 | 输出产物 | 输出 `final.mp4`、脚本、分镜、合规报告 | P0 |

### 5.2 MVP 暂不做

| 功能 | 不做原因 |
|---|---|
| 实时行情接入 | 容易引出荐股和实时交易建议风险 |
| 个股分析视频 | 内容边界复杂，合规风险高 |
| 自动买卖点生成 | 不符合“科普工具”定位 |
| 复杂指标计算 | 第一版用虚拟示意图即可 |
| 多语言 | 先聚焦中文短视频 |
| 横屏视频 | 先聚焦视频号 + 抖音竖屏 |

## 10. 视频结构设计

MVP 固定 7 段式结构：

| 段落 | 时间 | 目标 | 画面类型 |
|---|---:|---|---|
| Hook | 0-3s | 抓住注意力 | 大标题 + 强问题 |
| Misunderstanding | 3-8s | 纠正常见误区 | 对比卡片 |
| Concept | 8-18s | 讲清基本概念 | 指标结构图 |
| How It Works | 18-32s | 解释运行逻辑 | 动态示意图 |
| How To Use | 32-45s | 说明使用场景 | 三点卡片 |
| Pitfall | 45-55s | 提醒常见坑 | 风险提示卡 |
| Summary | 55-60s | 一句话总结 | 金句收尾 |

MACD 样例结构：

| 段落 | 画面标题 | 核心表达 |
|---|---|---|
| Hook | 金叉就能买？ | 先打破新手误区 |
| Misunderstanding | MACD 不是买卖按钮 | 指标只是辅助观察 |
| Concept | MACD 看的是趋势 | 解释 DIF、DEA、柱子 |
| How It Works | 快线慢线的距离 | 用红绿柱讲变化 |
| How To Use | 看位置 + 趋势 + 量能 | 不单看金叉死叉 |
| Pitfall | 高位金叉也可能是坑 | 强调场景差异 |
| Summary | MACD 看趋势，不预测未来 | 收束观点 |

## 11. 视觉风格设计

### 11.1 黑金交易室风

适用：MACD、KDJ、成交量、均线。

特点：

- 深色背景。
- 金色或暖色强调色。
- 红绿柱、折线、网格、K 线示意图。
- 类似交易终端，但避免太复杂。
- 信息密度中等，重点突出。

### 11.2 白底科普卡片风

适用：止损、支撑压力、交易纪律、新手误区。

特点：

- 白色或米白背景。
- 大标题 + 卡片解释。
- 少量图标和简化图形。
- 更适合视频号和中年用户。
- 克制、可信、非营销。

## 12. 文案规则

### 12.1 每屏文字限制

| 项目 | 限制 |
|---|---|
| 主标题 | 不超过 16 个中文字符 |
| 副标题 | 不超过 24 个中文字符 |
| 要点数量 | 不超过 3 个 |
| 单个要点 | 不超过 12 个中文字符 |
| 整屏总字数 | 建议不超过 45 字 |

### 12.2 口播规则

1. 使用大白话。
2. 少用复杂金融术语。
3. 必须解释术语含义。
4. 不输出具体买卖建议。
5. 不使用“稳赚”“必涨”“抄底”“梭哈”等诱导词。
6. 结尾必须包含风险提醒或学习提醒。

### 12.3 禁止话术

```text
这只股票可以买
明天大概率上涨
现在就是买点
闭眼买
稳赚不赔
翻倍机会
主力要拉升
赶紧上车
牛股来了
```

### 12.4 推荐话术

```text
这个指标只能作为观察工具
不能单独作为买卖依据
还要结合趋势、位置和成交量
新手更应该先学会风险控制
历史案例不代表未来表现
```

## 13. 合规要求

本功能必须以“金融知识科普”为边界，不得变成荐股工具。

| 检查项 | 风险说明 | 处理方式 |
|---|---|---|
| 个股推荐 | 直接引导买卖 | 阻断或改写 |
| 涨跌预测 | 暗示未来走势 | 阻断或改写 |
| 收益承诺 | 承诺收益或胜率 | 阻断 |
| 煽动交易 | 鼓励追涨杀跌 | 改写 |
| 夸张标题 | 类似荐股营销号 | 降级改写 |
| 缺少风险提示 | 误导用户 | 自动追加 |

默认风险提示：

```text
以上内容仅作知识科普，不构成投资建议。
```

更自然的口播版本：

```text
记住，指标只是工具，真正下决策前还要结合趋势、位置和风险承受能力。
```

## 15. 验收标准

### 15.1 功能验收

| 编号 | 验收项 | 标准 |
|---|---|---|
| A1 | 可输入主题 | 输入“60 秒带你搞懂 MACD”后可创建任务 |
| A2 | 可生成脚本 | 生成约 60 秒中文口播稿 |

### 15.2 内容验收

| 编号 | 验收项 | 标准 |
|---|---|---|
| C1 | 科普性 | 内容解释清楚，不空泛 |
| C2 | 新手友好 | 术语有解释，口播不晦涩 |
| C3 | 不荐股 | 不出现具体买卖建议 |
| C4 | 不预测 | 不出现确定性涨跌判断 |
| C5 | 有风险提示 | 结尾或画面包含风险提醒 |
| C6 | 画面清晰 | 每屏重点明确，不堆字 |

### 15.3 视觉验收

| 编号 | 验收项 | 标准 |
|---|---|---|
| V1 | 画幅 | 1080x1920 竖屏 |
| V2 | 字幕 | 移动端可读，不贴边 |
| V3 | 风格 | 黑金或白底风格统一 |
| V4 | 节奏 | 每 5-10 秒有明显画面变化 |
| V5 | 片尾 | 有总结或风险提醒 |

## 18. 最终结论

MVP 的核心不是做“最强炒股视频生成器”，而是先做一个稳定、克制、合规的“股票知识点科普视频生成器”。第一版只要把 MACD、KDJ、均线、成交量、支撑压力、止损这 6 类内容做稳定，就可以支撑持续内容生产。后续再考虑封面、批量生成、历史案例、模拟图表动画等增强能力。
