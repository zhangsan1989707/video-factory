GitHub 热榜速报设计审查与优化建议

设计问题总览
1
封面页对比度严重不足（图1）
浅米色背景 + 浅灰色文字，正文和说明文字几乎不可读。底部"本期重磅"卡片的副标题标签颜色过淡，在米色背景上几乎消失。
高优先
2
封面页留白过多，信息密度失衡（图1）
标题区之后有大量空白区域，像是内容未加载完成，视觉重心全部集中在上半部分，底部卡片孤立感强。
高优先
3
底部"本期重磅"卡片与背景融合（图1）
卡片背景色与页面背景过于接近，边界感弱。标签文字（JavaScript · agent-skills…）颜色太淡，读不清楚。
中优先
4
列表页项目卡片层级不清晰（图2）
第04、05条目缺乏火焰热度标识，视觉上比前三名"弱"，但缺乏设计意图说明。各卡片边框细节不一致（部分有彩色左边框，部分没有）。
中优先
5
详情页数据卡片排版可改善（图3）
Star数和热度并排的两列卡片留白略显局促，Fork数和主语言的卡片对比价值不高。热度趋势色块大小不统一，视觉上不整齐。
低优先

优化方向汇总
①
封面改深色背景 — 用深棕/深灰替换米色，金色标题对比度从约2:1提升至7:1以上，副标题、标签全部可读
②
消除封面大面积空白 — 将底部"本期重磅"卡片上移，或增加一个"本期亮点"横向滑动预览区
③
重磅卡片加强边界 — 卡片背景加深，或加金色左边框，让它从背景中"浮"出来
④
列表页统一卡片规则 — 明确热度等级与左边框颜色的映射关系，04/05如果是"稳步上升"可用蓝色边框区别于"爆发"的橙色
⑤
详情页热度趋势规范化 — 色块统一高度，最近7天用深色、更早用浅色，形成时间轴感
