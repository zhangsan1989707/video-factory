# GitHub 热榜竖屏模板 V3 设计

## 目标

把当前 GitHub 热榜竖屏视频从“科技感 PPT 信息卡”升级为“真实项目证据 + 高级榜单视觉 + 克制短视频节奏”的完整模板。

本次不只优化单个项目卡，也要统一优化开场、榜单总览、项目介绍、结尾和字幕。项目截图必须进入模板核心，因为没有真实画面时，观众很难判断项目到底长什么样。

## 当前问题

当前模板的问题不只是某一张项目卡：

- 画面多由文字构成，缺少项目截图或真实页面证据。
- 开场、榜单总览、项目卡、结尾的视觉层级不完全统一。
- 背景网格偏亮，容易抢主体内容。
- 黄色和绿色使用范围过大，导致重点不唯一。
- 字幕和画面信息有时重复，浪费短视频里的注意力。
- 口播链路虽然有 AI 调用入口，但模型不可用时会回退模板口播；用户需要明确知道当前是 AI 生成还是模板回退。

## 设计方向

整体风格靠近：

- SaaS 产品官网
- GitHub 年度榜单
- Apple 发布会式克制动效

核心原则：

- 真实截图优先，文字只负责提炼。
- 绿色只表示 Star。
- 黄色只表示排名。
- 背景退后，项目信息和截图前置。
- 画面展示结论，口播补充解释，不做简单重复。

推荐配色：

- 背景：`#021426`
- 网格：`#0B3550`，低透明度
- 项目名：`#F5F8FF`
- 正文：`#D7E3F5`
- 标签：`#7FA3C8`
- Star 强调色：`#2EF2C2`
- 排名强调色：`#FFD84D`
- 横线、圆点、弱分割：`#4A5D75`

## 截图策略

每个项目至少尝试准备一个真实视觉素材，优先级如下：

1. 项目 homepage 截图。
2. README 中的图片。
3. GitHub 仓库页截图。
4. 采集失败时才回退为纯信息卡。

截图不是装饰，而是证明。项目卡里应该出现截图区域，尺寸要足够让观众感知项目长相；如果截图质量差，再降低权重而不是强行铺满。

## 模板结构

### 开场

开场不再只做抽象标题。它要快速建立本期主题、项目数量和榜单感。

画面重点：

- GitHub 热榜标题。
- 本期项目数量。
- 一句选择标准，例如“只看真实可用的新项目”。
- 背景弱网格和轻微中心光。

动效：

- 标题淡入。
- 选择标准轻微上浮。
- 背景网格慢速漂移。

### 榜单总览

榜单总览要像“排名面板”，但不能像后台表格。

画面重点：

- Top 5 或 Top 10 项目名。
- Star 数作为辅助信息。
- 当前即将进入的项目有轻微高亮。

动效：

- 排名逐条出现。
- 当前项目高亮条轻微扫过。
- 不使用大量边框和厚重卡片。

### 项目介绍卡

项目卡是核心页。

画面结构：

- 项目名最大。
- Star 第二视觉，用绿色。
- 排名徽章用黄色，但只出现一次。
- 项目截图作为真实证据区。
- 三条要点压成轻量标签或短行，不做大卡片。

动效：

- 项目名淡入并轻微上浮。
- Star 淡入并轻微缩放。
- 排名徽章轻微弹出一次。
- 截图轻微放大或上移，模拟真实展示。
- 三条要点依次出现。

### 结尾

结尾要帮助观众回忆，而不是只喊互动。

画面重点：

- 本期最值得回看的 2-3 个项目。
- 一句问题式互动，例如“你想先看哪个项目实操？”。
- 低调 CTA。

动效：

- 项目名标签逐个出现。
- 结尾问题稳定停留，方便观众读完。

### 字幕

字幕要补充画面，不重复画面。

规则：

- 画面已经写项目名、Star、排名时，字幕不再逐字重复。
- 字幕优先解释“为什么值得看”和“适合谁”。
- 字幕框降低黑色压迫感，透明度和圆角保持克制。
- 字幕最多 2-3 行，避免盖住截图主体。

## AI 口播状态

控制台必须明确展示口播来源：

- AI 生成成功：显示供应商和模型。
- AI 生成失败：显示失败原因，并说明已回退模板。
- 模型未配置或未测试成功：在生成口播前后都要清楚提示。

这样用户不会误以为模板口播是 AI 口播。

## 实现范围

更新素材链路：

- 控制台热榜任务要为每个项目准备可截图素材。
- 优先使用 homepage、README 图片和 GitHub 仓库页。
- 保留失败回退，不能因为单个截图失败中断整条视频。

更新 `src/composer/vertical.py`：

- 统一 `_hotlist_bg` 的配色和背景动效。
- 优化 `hotlist_opening`。
- 优化 `hotlist_ranking`。
- 优化 `hotlist_rank_card`，并加入截图区域。
- 优化 `hotlist_closing`。
- 优化底部字幕样式。

更新控制台任务逻辑：

- 让 `asset_manifest.json` 中的项目素材可用于项目卡截图。
- 在任务详情、日志或模型调用状态中明确展示口播是否由 AI 生成。

更新测试：

- 覆盖项目素材 manifest 的生成。
- 覆盖截图失败时仍能回退。
- 覆盖口播 AI 成功、失败、跳过三种状态。
- 保留现有计划生成、预览帧和 dry run 测试。

## 验收标准

- 开场、榜单总览、项目卡、结尾和字幕形成统一视觉体系。
- 每个项目卡优先展示真实截图；没有截图时才使用纯信息卡。
- 绿色只用于 Star。
- 黄色只用于排名。
- 背景网格不抢主体。
- 字幕不遮挡截图主体，不和画面重复关键信息。
- 控制台能明确看出口播是 AI 生成还是模板回退。
- 现有控制台任务测试通过。
- `compileall` 和 `git diff --check` 通过。

## 不做范围

- 云端发布或自动发视频。
- 可视化拖拽时间线。
- 用户手动调色的主题编辑器。
- 为每个项目定制完全不同的版式。
