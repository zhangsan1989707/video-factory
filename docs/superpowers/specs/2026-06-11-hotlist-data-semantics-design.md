# Hotlist 数据口径澄清设计

## 目标

修正热榜控制台、发布辅助信息和热榜模板中对 `daily_growth` 的表达，明确它是估算热度，不是真实新增 star。

## 范围

只处理 `P0-3 明确热榜数据口径`：

- 候选数据与候选表展示
- 发布辅助包与准备度报告
- 热榜口播生成与脚本质检
- 热榜模板中的估算热度文案

## 当前问题

当前 `daily_growth` 是按当前总 stars 与仓库创建时间折算出的估算值，但多处文案仍可能让人误解为真实新增：

- 候选表未明确口径
- 发布辅助包和封面副标题会写“日均涨星”
- 热榜模板出现“新增星标”“热度爆发”
- 口播提示与质检没有显式禁止把估算值说成真实增长

## 设计

### 统一表达

- `daily_growth` 的显示文案统一为 `估算日均 star ...`
- 数据口径说明统一为：
  - `热度口径：估算日均 star 由当前总 stars 和仓库创建时间折算，不是真实新增 star。`

### 控制台

- 候选表在星标列直接显示 `daily_growth`
- 候选生成日志写入口径说明

### 发布辅助信息

- `publish_pack.json` 增加 `data_note`
- 发布描述中加入数据说明段落
- `readiness_report.json` 增加 `data_semantics` 检查项

### 口播与质检

- 口播生成 / 润色 prompt 明确禁止把估算热度写成真实新增 star
- 本地质检新增规则，识别：
  - `新增 star`
  - `今天涨了`
  - `单日新增`
  - `暴涨了`
  - 等真实增长口吻

### 热榜模板

- 总览与详情中的热度相关标签改成“估算热度”语义

## 验收标准

- 候选表能看到估算热度口径
- 发布辅助包或准备度报告中能看到数据来源/口径说明
- 脚本质检能识别“真实增长”类过度表述
- 对应测试通过

## 不做范围

- 不改 GitHub API 抓取策略
- 不引入真实 star 增量历史数据源
- 不重做热榜排序算法
