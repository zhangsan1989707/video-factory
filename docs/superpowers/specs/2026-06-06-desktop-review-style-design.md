# Desktop Review Style - 横屏项目讲解模式

## 目标

新增 `desktop-review` 风格，生成接近参考视频的横屏项目讲解画面：桌面背景、浏览器窗口、固定标题、底部大黄字、真实网页滚动、局部放大和鼠标读屏。

## 非目标

- 不复刻参考视频右上角 logo。
- 不替换当前竖屏 V2 流水线。
- 不做复杂剪辑器。
- 不保证每个网站都有定制交互。

## 输出规格

- 默认画幅：`1592x1080`
- 默认时长：约 30 秒
- 默认帧率：`30fps`
- 画面元素：
  - 顶部左侧红白标题
  - 顶部右侧账号文字占位
  - 中间深色浏览器窗口
  - 底部固定黄色视频标题
  - 大鼠标指针
  - 局部高亮/放大

## 分镜结构

1. 仓库总览：打开 GitHub repo，给出推荐结论。
2. 项目价值：放大 About/README 标题区。
3. 证据一：滚动到 README 图片或产品截图。
4. 证据二：滚动到功能段落。
5. 证据三：滚动到安装/代码/配置段落。
6. 收尾：回到仓库或 stars 区，给出关注理由。

## 实现方式

- `planner/desktop_review.py` 生成 `desktop_review_plan.json`。
- `browser/desktop_review_recorder.py` 用 Playwright 抓取分镜帧。
- `composer/desktop_review.py` 合成桌面风格视频。
- CLI 新增 `--style desktop-review`。

## 验收标准

- 成片不是卡片式静态讲解，而像真实屏幕浏览。
- 每 2-5 秒有明显画面状态变化。
- 鼠标停在被讲解区域附近。
- 底部大黄字固定表达视频主题。
- GitHub/README 页面至少出现 3 种不同局部：仓库总览、README 视觉素材、功能/代码段落。
