# 定时生产独立页面与全自动出片设计

## 背景

当前定时能力藏在配置抽屉的“定时草稿”区，只支持到候选草稿或口播草稿。用户希望每天早上无人确认地生成一个正式 mp4。定时生产会继续增加时间、榜单、模板、BGM、质量门和运行状态等参数，因此不再适合放在基础配置抽屉里。

## 目标

- 新增独立的“定时生产”入口，用于配置和查看无人值守生产计划。
- 支持每天或每周一自动生成 GitHub 热榜正式 mp4。
- 保留现有候选草稿、口播草稿模式，新增正式视频模式。
- 不做自动发布、通知、多计划、多生产线调度。

## 产品设计

主界面顶部操作区新增“定时生产”一级入口，打开独立的定时生产视图。设置抽屉继续只负责 GitHub、模型供应商和任务路由等基础配置。

定时生产页展示这些区域：

- 计划状态：启用开关、上次运行、下一次运行、最近失败原因。
- 自动化级别：只生成候选草稿、自动生成口播草稿、自动生成正式视频。
- 执行计划：每日或每周一、执行时间。
- 内容参数：榜单窗口、项目数。
- 视频参数：视觉风格、渲染引擎、字幕模式、口播语气、BGM、BGM 音量、BGM 路径。
- 最近定时任务：列出 scheduled 任务中待处理、失败和完成项，可点击进入任务详情。
- 操作：保存计划、立即试跑当前计划。

## 后端设计

`scheduler.mode` 增加 `auto_video`：

- `candidates_only`：保持旧行为，只生成候选草稿。
- `auto_script`：保持旧行为，自动确认前 N 个候选并生成口播草稿。
- `auto_video`：自动完成候选、选择、口播确认、计划生成、计划校验、视频渲染和正式 mp4 生成。

`auto_video` 使用现有 GitHub 热榜任务链路，不新增并行实现。流程为：

1. 创建 scheduled `github_hotlist` 任务。
2. 调用 `generate_candidates(job_id)`。
3. 选择前 `project_count` 个候选。
4. 调用 `save_selection(job_id, {"items": selected})` 生成口播。
5. 调用 `save_script(job_id, {"segments": segments})` 确认口播。
6. 调用 `prepare_plan(job_id)`。
7. 调用 `validate_plan(job_id)`。
8. 调用 `render_video(job_id)`，最终任务进入 `completed` 并生成正式 mp4。

质量门不自动忽略。若 `save_script` 因质检阻断而停在 `awaiting_script_confirmation`，定时任务视为失败，保留质量报告、日志和任务快照，不更新 `last_run_date`。

失败时不更新 `last_run_date`，以便下一次调度继续重试。防重复运行继续沿用当前 `_RUNNING_KEYS` 机制。

## 数据与兼容性

现有 `scheduler.json` 继续兼容。新增模式只扩展枚举，不迁移旧字段。

`template_params` 继续保存视频参数，并沿用现有标准化逻辑。定时生产页保存时写入同一个 `scheduler` 配置，避免增加第二套配置来源。

## 前端实现范围

- 从设置抽屉移除定时草稿编辑区，保留一个指向“定时生产”的入口提示。
- 新增定时生产视图的 DOM、样式和渲染函数。
- `renderScheduler` 支持 `auto_video` 的文案和状态。
- `saveSettings` 不再负责 scheduler；新增定时生产页保存逻辑写入现有批量配置 API。
- 最近定时任务复用现有历史任务数据和 `scheduled` 标记。

## 验收标准

- 用户可以从独立入口进入定时生产页，配置不再只藏在基础配置抽屉里。
- 可以保存并恢复 `auto_video`、执行时间、榜单窗口、项目数和视频参数。
- 到点后 `auto_video` 能生成 `completed` 任务，并产出正式 mp4 文件。
- 候选草稿和口播草稿模式行为不变。
- 自动出片失败时任务保留失败阶段和日志，不更新 `last_run_date`。
- 质检阻断不会被自动忽略。
- 前端测试覆盖定时生产页渲染、保存 payload 和模式文案。
- 后端测试覆盖成功自动出片、失败不标记已运行、旧模式不回归。

## 非目标

- 不支持多个定时计划。
- 不支持自动发布到社交平台。
- 不支持通知、邮件、飞书消息。
- 不支持按节假日或复杂 cron 表达式运行。
- 不支持定时生成非 GitHub 热榜任务。
