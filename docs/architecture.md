# GitHub Video Maker 架构设计文档

## 概述

GitHub Video Maker 是一个 AI 驱动的 GitHub 项目介绍视频自动生成工具，主要面向中文短视频平台（抖音/快手）。系统采用模块化设计，支持多种视频风格和渲染引擎。

## 核心模块

### 1. 流程协调器 (src/pipeline.py)

负责协调整个视频生成流程，支持三种工作模式：

#### 1.1 从计划文件继续 (from-plan)
```
输入: shot_plan.json 或 desktop_review_plan.json
  ↓
读取分镜 → 生成脚本 → TTS → 录制/采集素材 → 合成视频 → 后处理
  ↓
输出: final.mp4
```

#### 1.2 热榜模式 (多 URL)
```
输入: 多个 GitHub URL (逗号分隔)
  ↓
抓取项目信息 → 生成热榜分镜 → 生成脚本 → 采集素材 → TTS → 合成竖屏视频 → 后处理
  ↓
输出: final.mp4
```

#### 1.3 单项目模式
```
输入: 单个 GitHub URL
  ↓
┌─ desktop-review: 生成桌面审阅分镜 → TTS → 录制浏览器 → 合成桌面风格视频
├─ 竖屏: 生成 brief → 素材清单 → 分镜 → 脚本 → 采集素材 → TTS → 合成竖屏视频
└─ 横屏: 生成脚本 → TTS → 录制浏览器 → 鼠标动效 → 合成横屏视频
  ↓
输出: final.mp4
```

### 2. 数据模型 (src/models.py)

定义了所有核心数据结构：

```python
ProjectInfo        # GitHub 项目信息

AssetManifest      # 素材清单
VisualAsset        # 视觉素材
ProjectPaths       # 项目输出路径
```

#### 3.1 架构

#### 3.2 任务工作流

```
创建任务 → 生成候选 → 选择项目 → 确认脚本 → 生成计划 → 校验 → 渲染 → 最终化
   │          │          │          │          │        │       │       │
   ▼          ▼          ▼          ▼          ▼        ▼       ▼       ▼
 GH-xxx   AI 分析    AI 钩子    AI 质检   shot_plan  dry_run  渲染   编号文件
           AI 排序    AI 口播    发布包
```

### 3. 视频风格

#### 3.1 横屏风格 (desktop-review)
- 分辨率: 1592x1080
- 特点: 桌面浏览器风格，带开场卡、桌面背景、浏览器窗口、鼠标指针
- 流程: 浏览器录制 → 鼠标动效 → 合成

#### 3.2 竖屏风格 (vertical)
- 分辨率: 1080x1920
- 特点: 适合短视频平台，深色科技风
- 渲染引擎:
  - **PIL 渲染**: 逐帧渲染，15+ 种视觉模板
  - **HyperFrames 渲染**: HTML 动画 → MP4

#### 3.3 热榜风格 (hotlist)
- 分辨率: 1080x1920
- 特点: 多项目排行榜展示
- 模板: tech_hotspot 风格

### 4. 渲染引擎

#### 4.1 PIL 逐帧渲染 (composer/vertical.py)
```
优点: 完全 Python 实现，无外部依赖
缺点: 渲染速度较慢，动画效果有限
适用: 简单静态画面
```

#### 4.2 HyperFrames 渲染 (hotlist_v2/render.py)
```
优点: HTML/CSS 动画，效果丰富，支持复杂交互
缺点: 需要 Node.js 环境
适用: 科技热点风格视频
```

## AI 模型路由

### 支持的供应商

| 供应商 | API 类型 | 模型示例 |
|--------|----------|----------|
| OpenAI | OpenAI 兼容 | gpt-4, gpt-3.5-turbo |
| Anthropic | Anthropic API | claude-3-opus, claude-3-sonnet |
| DeepSeek | OpenAI 兼容 | deepseek-chat |
| 小米 | OpenAI 兼容 | mimo-v2.5-pro |

### AI 任务类型

| 任务 | 用途 | 输出格式 |
|------|------|----------|
| candidate_analysis | 候选项目分析 | JSON |
| hotlist_ranking | 热榜排序 | JSON |
| hook_generation | 标题钩子生成 | JSON |
| narration_generation | 口播脚本生成 | JSON |
| script_polishing | 脚本润色 | JSON |
| fact_check | 脚本质检 | JSON |

### 防御性设计

所有 AI 调用都有 fallback 机制：
- **AI 路径**: 调用 AI API 获取结果
- **Fallback 路径**: 使用启发式规则或默认模板
- **保证**: 即使 AI 失败，流程也不会阻塞

## 安全设计

### 1. API Key 保护
- 配置显示时脱敏：只显示前 4 位和后 4 位
- 存储时合并：防止前端覆盖已保存的密钥

### 2. 路径安全
- 符号链接防护：拒绝符号链接目录
- 路径遍历防护：验证所有路径在允许范围内

### 3. 输入验证
- GitHub URL 格式验证
- 文件名安全字符检查
- 参数类型和范围验证

## 扩展性

### 添加新视频风格
1. 在 `src/planner/` 添加分镜生成器
2. 在 `src/composer/` 添加合成器
3. 在 `src/pipeline.py` 添加风格路由

### 添加新渲染引擎
1. 实现渲染函数（输入: 分镜 + 素材，输出: 视频文件）
2. 在 `hotlist_v2/render.py` 或 `composer/` 中集成
3. 通过 `render_engine` 配置切换

### 添加新 AI 供应商
1. 在 `model_router.py` 实现 API 调用
2. 在 `providers.json` 添加配置
3. 在 `model_routing.json` 添加路由
